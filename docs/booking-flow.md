# 예약 플로우 & 환불 규정 (docs/booking-flow.md)

고객 예약 진행 흐름과 환불(취소) 규정 정의. schema.md의 상태 머신·결제 정합성 원칙과 함께 본다.

## 고객 예약 플로우

```
[1] 슬롯 선택  →  [2] 정보 입력  →  [3] 결제  →  [4] 완료/조회
```

### [1] 슬롯 선택
- 고객이 날짜(금/토)와 시설 종류, 부(1부/2부)를 선택.
- 화면에는 예약 가능한(slots.status = 'open') 슬롯만 노출. 'booked'/'closed'는 선택 불가 표시.
- 시설 종류별 고정가를 함께 표시.
- 선택 시점에는 아직 슬롯을 잠그지 않음(둘러보기 단계). 잠금은 [3] 결제 진입 시.

### [2] 정보 입력
- 비회원 예약: 예약자명, 연락처(휴대폰), 인원 입력.
- 연락처는 필수 — 문자 알림 및 예약 조회 수단.
- 입력값 검증(휴대폰 형식 등)은 클라이언트 + 서버 양쪽.
- 환불 규정·이용 안내 동의 체크(분쟁 예방).

### [3] 결제
- 서버가 트랜잭션으로 슬롯 선점: slot을 FOR UPDATE 잠금 → status 'open' 확인 → bookings를 `pending_payment`로 생성 → slot을 'booked'로 변경.
- 선점 성공 후 토스페이먼츠 결제창 호출(전액 선결제).
- 결제 성공 콜백 → **서버에서 토스 결제 승인 API로 금액·주문ID 재검증** → payments 기록, bookings를 `confirmed`로 갱신(트랜잭션).
- 결제 실패/이탈/타임아웃(기본 10분) → bookings `cancelled`, slot 'open' 복구.
- 확정 시 예약 확정 문자 발송(알림 인터페이스 경유).

### [4] 완료/조회
- 완료 화면에 예약번호(booking_number) 표시.
- 예약 조회: **예약번호 + 연락처** 조합으로 서버에서 검증 후 조회(회원 로그인 없음).
- 조회 화면에서 예약 상태 확인 및 취소 요청 가능.

## 환불 / 취소 규정

이용일 기준 취소 시점에 따라 환불 금액이 달라진다. **결제 금액(bookings.amount) 기준.**

| 취소 시점 | 차감 | 환불액 |
|-----------|------|--------|
| 이용 2일 전까지 | 0% | 결제금액 100% 환불 |
| 이용 1일 전 | 50% | 결제금액 50% 환불 |
| 이용 당일 | 100% | 환불 없음 |

### 기준 시점 계산 (분쟁 방지)
- 기준은 **이용일 자정(해당 날짜 00:00, KST)**.
- `days_until = 이용일(00:00) − 취소요청 시각`을 날짜 단위로 계산.
  - 이용일까지 **2일 이상** 남음 → 전액(100%) 환불
  - 이용일까지 **1일** 남음 → 50% 환불
  - 이용일 **당일**(0일) → 환불 없음
- 예) 토요일 이용 건:
  - 목요일 23:59까지 취소 → 100% 환불
  - 금요일 중 취소 → 50% 환불
  - 토요일 중 취소 → 0% 환불
- 시간대는 KST(Asia/Seoul) 고정. 서버에서 계산하며 클라이언트 시간 신뢰하지 않음.
- 환불율은 정책 상수/테이블로 분리해 관리자가 추후 조정 가능하게 둔다(초기엔 위 값 하드코딩 가능, 단 한 곳에서 관리).

### 환불 처리 정합성 (schema.md 재확인)
- 반드시 **토스 결제 취소 API 호출 성공 → DB 상태 갱신** 순서.
- 부분 환불(50%)은 토스 부분 취소 사용, payments.status = 'partial_cancelled'.
- 환불 완료 후 bookings `refunded`, slot 'open' 복구, 취소 문자 발송.
- 관리자도 /admin에서 동일 로직으로 수동 취소·환불 가능.

## 미정 / 추후 반영
- pending_payment 타임아웃 확정값(기본 10분 제안)
- 슬롯 자동 생성 주기(몇 주치 미리 open 할지)
- 환불 규정 동의 문구 최종본
- 노쇼(취소 없이 미방문) 처리: 당일 취소와 동일하게 환불 없음으로 간주
